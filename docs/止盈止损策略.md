## 1. 设计目标与原则

**目标**：用机械化规则替代情绪化决策，做"交易纪律哨兵"——到价提醒，截断亏损、让利润奔跑。

**核心原则**：
- **纯提醒**：App 不直接下单，只提醒；平仓由用户在券商端操作后回 App 确认。
- **多层保护**：硬止损（兜底）+ ATR 止损（核心）+ 保本止损（利润保护）三层止损，取更紧者。
- **防插针**：止损需维持 N 分钟确认，避免瞬时插针误触。
- **防漂移**：保本止损的风险额度用进场初始止损算，不随 ATR 波动漂移。
- **锁定利润**：移动止盈设利润锁定下限，不因高波动回撤吐光利润。

## 2. 整体架构

### 评估流程

每 15 秒一个监控 tick，对每只未了结持仓调用 `StrategyEvaluator.evaluate()`：

```
tick(position)
  → 拉行情（现价 + 日K）
  → evaluate(klines, currentPrice)
      → 用建仓日及之后的日K最高价刷新 highestPrice
      → 算 ATR
      → 算止损线（硬止损 + ATR + 保本，取更紧）
      → 算止盈线（分批目标价 或 移动止盈）
      → 触发检测（止损/止盈/保本告知）
      → 返回 stopPrice, takeProfitPrice, alert
  → 更新 position.stopPrice / takeProfitPrice
  → 若有 alert，插入提醒列表
```

### 无状态评估器

`StrategyEvaluator` 每 tick **重新构造、无状态**。所有跨 tick 的状态（初始止损、保本阶段、破位计时、分批档数）存在 `Position` 对象上，评估器读写这些字段。

### 纯提醒模型

App 只提醒不操作。用户确认提醒时，按提醒类型差异化处理（见 [§9](#9-纯提醒模型与确认处理)）。

## 3. ATR 计算

**算法**：Wilder 平滑法（`lib/core/strategy/atr.dart`）。

**真实波幅 TR**（每根日K）：
```
TR = max(当日最高 − 当日最低, |当日最高 − 前收|, |当日最低 − 前收|)
```
含跳空缺口。

**ATR 平滑**：
- 首周期：前 `period` 个 TR 的简单平均
- 后续：`ATR = (前ATR × (n−1) + 本TR) / n`（Wilder 平滑）

**输入**：最近 30 根日K（升序），`period` 由策略配置（默认 14，趋势保守用 22）。

**返回**：最近一根 K 线对应的 ATR（价格单位）。数据不足（<2 根）返回 null。

**数据不足处理**：
- TR 数 < period：返回已有 TR 的简单平均（近似）
- TR 数 ≥ period：标准 Wilder 平滑

**复权口径**：K 线用前复权（`fqt=1` / `qfq`），消除除权缺口。已知限制：前复权基准与未复权成本价在有除权的股票上不一致，影响 ATR 止损精度（低频，TODO）。

## 4. 止损线计算（三层）

```
止损线 = max( 硬止损, ATR止损, 保本止损线 )
```
取**更高（更紧）**者——更高的止损线 = 更早触发 = 亏得更少。

### 4.1 第一层：硬止损（兜底）

```
hardStop = 成本 × (1 − hardStopPercent)
```

- 永远存在，不可关闭。
- 三个预设：10%、12%、10%。
- 作用：灾难性兜底。即使 ATR 算不出（数据不足），止损仍有效。

### 4.2 第二层：ATR 止损（核心）

```
atrStop = 成本 − ATR × atrMultiple
```

**固定倍数**（`atrMultiple`，三预设不同）：
- 趋势短线：2.0（跟紧，止损快）
- 波段：2.5（标准）
- 中线：3.0（给趋势空间，容忍回撤）

ATR 本身已随股票波动自适应（高波动 ATR 大、止损自动宽），倍数无需再叠加调整。

> **关于 ATR 自适应（已弃用）**：曾实现按波动率分档覆盖倍数（低波动2.0/中2.5/高3.5），
> 但"高波动给大倍数"不合理——ATR 已放大，倍数再放大导致高波动票止损距离双重放大、
> 单笔亏损过大（违背截断亏损）。故改为固定倍数。自适应逻辑代码保留（`atrAdaptive` 开关），
> 三预设默认关闭，不启用。

**基础止损**：
```
baseStop = max(hardStop, atrStop)
```

### 4.3 第三层：保本止损（利润保护）

**目的**：浮盈达标后，止损线上移到成本以上，锁定利润，避免盈利单变亏损单。

**初始止损**（进场首次记录，防漂移）：
```
if (position.initialStopPrice == null && atr != null):
    position.initialStopPrice = baseStop   // 首次的 max(硬止损, ATR止损)
```
用进场初始止损算风险额度，不随后续 ATR 波动漂移。

**风险额度**（每股）：
```
riskPerShare = 成本 − initialStopPrice
```

**浮盈倍数**：
```
profitMultiple = (现价 − 成本) / riskPerShare
```

**阶段判定**（按 `breakevenStages` 升序，默认三阶段）：

| 阶段 | 触发条件 | 止损线上移到 | lockRatio |
|------|----------|--------------|-----------|
| 1（保本） | 浮盈 ≥ 1×风险额度 | 成本 + 0×风险 = **成本** | 0.0 |
| 2（锁半利） | 浮盈 ≥ 2×风险额度 | 成本 + 0.5×风险 | 0.5 |
| 3（锁70%利） | 浮盈 ≥ 3×风险额度 | 成本 + 0.7×风险 | 0.7 |

**阶段只升不降**：
- `targetStage` = 当前浮盈达到的最高阶段
- 若 `targetStage > position.breakevenStageReached`：更新 `breakevenStageReached`，发保本告知提醒
- 用 `breakevenStageReached`（历史最高）算线，**浮盈回落后止损线不回撤**

**保本止损线**：
```
if (breakevenStageReached > 0):
    breakevenStopPrice = 成本 + reachedStage.lockRatio × riskPerShare
```

### 4.4 最终止损线与触发类型

```
stopPrice = max(baseStop, breakevenStopPrice)
```

**触发类型判定**（跌破时告警标哪种）：
- 保本线更紧（`breakevenStopPrice > baseStop`）→ `breakevenStop`
- 否则硬止损更紧（`hardStop ≥ atrStop`）→ `hardStop`
- 否则 → `atrStop`

## 5. 止盈线计算（两类）

按 `takeProfitStrategy` 选择：

### 5.1 分批止盈（batchAndTrailing，仅波段预设）

**目标价**（下一未触发档）：
```
riskPerShare = 成本 − initialStopPrice
takeProfitPrice = 成本 + nextTarget.riskRewardRatio × riskPerShare
```

默认两档：
- 第1档：盈亏比 2:1（到价卖 40%）
- 第2档：盈亏比 4:1（到价卖 30%）
- 剩余 30%：转移动止盈

**档位推进**：`triggeredTpCount` 记录已触发档数，到价后递增，跳过已触发档取下一档。全档触发后，剩余仓位用移动止盈兜底。

### 5.2 钱德勒移动止盈（trailingOnly，或分批全触发后兜底）

**highestPrice 的取值**（关键）：
- 持仓期间最高价，**只取建仓日及之后的日K最高价**（`date >= createdAt`），不含建仓前的历史高点
- 每 tick 用日K区间最高价刷新：`highestPrice = max(highestPrice, max(建仓后日K的 high))`
- 日K high 是交易所真实日内最高，**不依赖 App 是否常开**——App 关闭期间漏掉的 tick 不影响，下次打开拉日K自动补上
- 只增不减，浮盈回落后 highestPrice 不回撤
- 初始值 = 录入时填的买入价（成本）

**裸值**：
```
raw = highestPrice − ATR × trailingMultiple
```
从持仓期间最高价往下减，回撤到此价才卖（右侧出场）。

**利润锁定下限**（关键，防止高波动吐光利润）：
```
realizedGain = highestPrice − 成本          // 已实现最高浮盈
lockedProfit = max(realizedGain × 0.6, 成本 × 0.08)   // 至少锁60%浮盈或8%利润
tpFloor = max(止损线 + ATR×0.5, 成本 + lockedProfit)
takeProfitPrice = max(raw, tpFloor)
```

**保证**：
1. 止盈永远高于止损线（`tpFloor ≥ 止损线 + 半个ATR`）
2. 已盈利时至少锁定六成浮盈或 8% 利润，不被高波动回撤吐光
3. 不与保本止损重合（保本止损线=成本时，止盈下限=成本+锁定利润 > 成本）

**未盈利不设移动止盈**：
```
if (realizedGain ≤ 0): takeProfitPrice = null   // 不设线，避免建仓即误触发
```

### 5.3 两者关系

- 波段预设：先分批止盈（目标价在上方，到价卖一部分），全档触发后剩余用移动止盈（回撤才卖）。
- 趋势预设：纯移动止盈，让利润奔跑，回撤才出场。

## 6. 触发检测与告警

每个 tick 评估后，若 `detect=true`（盘中），按优先级检测触发，**同一 tick 只产 1 条告警**：

### 优先级

1. **止损触发**：`现价 ≤ 止损线` → 走 5 分钟确认 → 确认通过则止损告警
2. **保本阶段提升告知**：浮盈达新阶段、止损线上移 → 告知性提醒（非紧急）
3. **止盈触发**：
   - 分批档到价：`现价 ≥ 分批目标价` → 分批止盈告警
   - 移动止盈到价：`现价 ≤ 移动止盈线` → 移动止盈告警

### 逻辑

```
if (现价 ≤ 止损线):
    if (确认通过): 产生止损告警
    else: 保留 stopBreachSince（确认中，不告警）
else:
    清空 stopBreachSince（回升脱险）
    if (保本阶段提升): 产生保本告知告警
    else if (止盈到价): 产生止盈告警
```

### 告警类型（AlertType）

| 类型 | 含义 | 确认时行为 |
|------|------|-----------|
| `hardStop` | 硬止损触发 | 全平（handled） |
| `atrStop` | ATR 止损触发 | 全平（handled） |
| `breakevenStop` | 保本阶段提升告知 | 仅已读（不平仓） |
| `takeProfitTarget` | 分批止盈档到价 | 按档累加 closedQuantity |
| `trailingTakeProfit` | 移动止盈触发 | 全平（handled） |
| `chandelierStop`/`structuralStop`/`timeStop` | 预留未实现 | — |

## 7. 止损维持 N 分钟确认

**目的**：过滤瞬时插针，避免一根 K 线插针就误触止损。

**机制**（`_checkStopConfirmation`）：
```
if (immediate || confirmMinutes ≤ 0): 直接通过
if (position.stopBreachSince == null): position.stopBreachSince = now
return now − stopBreachSince ≥ confirmMinutes
```

- 首次跌破：记录 `stopBreachSince`，不触发
- 持续在止损线下方满 `confirmMinutes`（默认 5 分钟）：触发
- 期间回升（现价 > 止损线）：清空 `stopBreachSince`，重新计时
- `immediate=true`（人工输价）：绕过确认立即触发

**默认 5 分钟**（`stopConfirmMinutes`，可配置，0=关闭确认）。

## 8. 盘后处理

**盘后（非交易时段）**：`detect=false`。
- **仍算止损止盈线**（让 UI 有数据展示，显示"已收盘"标）
- **不检测触发**（不产生告警）
- **不推进确认计时**（清空 `stopBreachSince`，避免盘后收盘价在止损线下方时"确认中"常驻）

**交易时段判断**：北京时间 9:30-11:30 / 13:00-15:00，周末休市。节假日未处理（TODO）。

## 9. 纯提醒模型与确认处理

App 只提醒不操作。用户在提醒页确认时，按类型差异化（`AppStore.confirmAlert`）：

| 告警类型 | 确认行为 |
|---------|----------|
| `hardStop` / `atrStop` / `trailingTakeProfit` | `handled = true`（持仓转"已平仓"，监控停止） |
| `takeProfitTarget`（分批止盈） | `closedQuantity += 总量 × sellRatio`（按档累加，不平仓，剩余继续监控；剩余为0则 handled） |
| `breakevenStop`（保本告知） | 仅标记已读（`handled` 保持 false） |

**分批止盈档位对齐**：`triggeredTpCount` 在评估器到档时已 +1，确认时取档索引 = `triggeredTpCount − 1`，据此取 `sellRatio`。

**撤销确认**：`reopenPosition` 将 `handled = false`，恢复监控（误确认时用）。

## 10. 跨 tick 状态

存在 `Position` 上的动态字段（评估器每 tick 读写）：

| 字段 | 类型 | 语义 |
|------|------|------|
| `highestPrice` | double | 持仓期间最高价（建仓日及之后，只增不减），用于移动止盈。每 tick 用「建仓日及之后的日K最高价」刷新，不依赖 App 是否常开 |
| `initialStopPrice` | double? | 进场初始止损（首次记录），用于保本风险额度，防漂移 |
| `breakevenStageReached` | int | 已达到的最高保本阶段（0-3），只升不降 |
| `stopBreachSince` | DateTime? | 止损首次破位时间，用于 N 分钟确认 |
| `triggeredTpCount` | int | 分批止盈已触发档数 |
| `closedQuantity` | int | 已平仓数量（分批止盈累加） |
| `handled` | bool | 是否已了结（确认止损/移动止盈后） |
| `stopPrice` | double? | 当前止损线（评估器更新） |
| `takeProfitPrice` | double? | 当前止盈线（评估器更新） |

**重置时机**：
- 加仓（`addFill`）/ 编辑成本（`updatePosition`）：成本变了，重置 `initialStopPrice`、`breakevenStageReached`、`triggeredTpCount`、`stopBreachSince`（基准已变，旧进度失效）
- `updatePosition` 还重置 `highestPrice = 新成本`

## 11. 预设策略

`StrategyConfig.fromPreset`：

| 预设 | 硬止损 | ATR周期×倍数 | 移动止盈倍数 | 止盈策略 | 适用 |
|------|--------|--------------|--------------|----------|------|
| 波段 | 10% | 14日 × 2.5 | ×3.0 | 分批+移动 | 波段操作，赚一段就走 |
| 中线 | 12% | 22日 × 3.0 | ×3.0 | 纯移动 | 看好趋势，容忍回撤 |
| 趋势短线 | 10% | 14日 × 2.0 | ×2.5 | 纯移动 | 止损紧、落袋快 |

> 预设 enum 名仍为 `trendConservative`/`trendAggressive`/`swingStandard`（持久化存 index，未改），显示名为中线/趋势短线/波段。

**三预设都开启**：保本止损、5 分钟确认（ATR 自适应已关闭，用固定倍数）。

**参数差异解读**：
- 硬止损：波段10%、趋势短线10%（紧）、中线12%（松，给趋势空间）
- ATR 倍数：趋势短线2.0（跟得紧）、波段2.5、中线3.0（宽，容忍回撤）
- ATR 周期：中线22日（更长更平滑稳定），其余14日（标准，更敏感）
- 移动止盈倍数：趋势短线2.5（紧，早落袋），其余3.0（给利润空间）

## 12. 完整计算示例

### 示例：东山精密（002384），中线预设

**录入**：成本 222.27，买入 2026-07-01。
**行情**（2026-07-17）：现价 241.92，30日 ATR ≈ 22.5，30日最高 280.08。

**步骤 1：ATR 止损（固定倍数 3.0）**
```
atrStop = 222.27 − 22.5 × 3.0 = 222.27 − 67.5 = 154.77
```

**步骤 2：硬止损 + 基础止损**
```
hardStop = 222.27 × (1 − 0.12) = 195.60
baseStop = max(hardStop, atrStop) = max(195.60, 154.77) = 195.60
```
（硬止损更紧，起作用的是硬止损）

**步骤 3：保本止损**
```
initialStopPrice = 195.60（首次记录）
riskPerShare = 222.27 − 195.60 = 26.67
profitMultiple = (241.92 − 222.27) / 26.67 = 19.65 / 26.67 = 0.74
```
浮盈 0.74 倍 < 1 倍 → 未达保本阶段，`breakevenStopPrice = null`。

**止损线** = max(195.60, null) = **195.60**

**步骤 4：移动止盈**
```
realizedGain = 280.08 − 222.27 = 57.81（已盈利）
lockedProfit = max(57.81 × 0.6, 222.27 × 0.08) = max(34.69, 17.78) = 34.69
tpFloor = max(195.60 + 22.5×0.5, 222.27 + 34.69) = max(206.85, 256.96) = 256.96
raw = 280.08 − 22.5 × 3.0 = 280.08 − 67.5 = 212.58
takeProfitPrice = max(212.58, 256.96) = 256.96
```
（裸值212.58低于下限，取下限256.96，锁定34.69元利润=15.6%）

**结果**：
- 止损线 **195.60**（成本下方12%）
- 止盈线 **256.96**（锁定15.6%利润，低于现价等回撤18.5元触发）
- 止盈 > 止损 ✓，止盈 > 成本 ✓（锁定利润）

### 示例：浮盈达保本阶段

若现价涨到 250（浮盈 27.73 = 1.04×风险额度）：
- `profitMultiple = 1.04 ≥ 1` → 阶段1（保本）
- `breakevenStopPrice = 222.27 + 0×26.67 = 222.27`（成本）
- 止损线 = max(195.60, 222.27) = **222.27**（上移到成本，保本）

若现价涨到 275（浮盈 52.73 = 1.98×风险）→ 仍阶段1（未达2倍）。
若现价涨到 276（浮盈 53.73 = 2.01×风险）→ 阶段2：
- `breakevenStopPrice = 222.27 + 0.5×26.67 = 235.61`（锁半利）
- 止损线 = **235.61**

之后回落到 250：`breakevenStageReached` 仍为2，止损线仍 235.61（不回撤）。

## 13. 边界与异常处理

| 场景 | 处理 |
|------|------|
| ATR 算不出（K线<2根） | `atr=null`，止损退回纯硬止损，止盈不设线 |
| ATR 数据不足一个周期 | 返回已有 TR 简单平均（近似） |
| 未盈利（highestPrice ≤ 成本） | 不设移动止盈线（null），避免建仓误触发 |
| 保本阶段浮动 | 阶段只升不降，用历史最高阶段算线 |
| 止损确认中 | 显示"确认中"（盘中），盘后清空不显示 |
| 盘后 | 算线但不触发，清空破位计时 |
| 已了结（handled） | 监控跳过，不再评估 |
| 加仓/编辑成本 | 重置保本阶段、初始止损、分批档数（基准变） |
| 止盈 < 止损（倒挂） | 移动止盈下限保证止盈 > 止损 + 半ATR，不会倒挂 |

## 15. 调优指南

### 可调参数

| 参数 | 位置 | 默认 | 调优方向 |
|------|------|------|----------|
| `hardStopPercent` | 预设 | 10-12% | 调宽让ATR主导，调紧加强兜底 |
| `atrMultiple` | 预设 | 2.0-3.0 | 大=容忍回撤，小=跟得紧 |
| `atrAdaptive` | 预设 | true | 关闭则用固定倍数 |
| `atrPeriod` | 预设 | 14/22 | 大=更平滑稳定，小=更敏感 |
| `trailingMultiple` | 预设 | 2.5-3.0 | 大=给利润空间，小=早落袋 |
| `breakevenStages` | 配置 | 1/2/3倍 | 调整保本触发阈值与锁定比例 |
| `stopConfirmMinutes` | 配置 | 5 | 0=关闭确认，大=更抗插针但延迟 |
| 移动止盈锁定比例 | 评估器 | 60%/8% | `realizedGain×0.6` 和 `cost×0.08` |
| `atrAdaptive` | 预设 | false | 自适应已弃用（见 §4.2），三预设关闭 |

### 调优建议

- **频繁被洗出**：增大 ATR 倍数、或增大确认时长
- **止损过晚亏损大**：调紧硬止损、或减小 ATR 倍数
- **止盈太早没吃到趋势**：增大移动止盈倍数、或调小锁定比例
- **止盈太晚吐光利润**：调大锁定比例（如 70%）、或减小移动止盈倍数
- **保本触发太早**：增大 `breakevenStages` 的 `riskMultiple`（如 1.5/3/4.5）
- **插针误触**：开启/增大 `stopConfirmMinutes`

### 注意事项

- ATR 用前复权 K 线，成本价未复权，除权股票上精度受影响（TODO）
- 5 分钟确认基于 15 秒 tick 计时，无分钟级 K 线接口
- 保本止损的 `initialStopPrice` 进场首次记录，若进场即大跌（ATR未稳定）可能偏高，只影响保本触发时机，不影响止损本身
- 纯提醒模型下，用户不确认则仓位不会了结，监控持续（去重避免重复告警）
